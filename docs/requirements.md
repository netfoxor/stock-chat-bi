# 股票数据分析系统 · 需求说明书

> 版本：v1.0  
> 日期：2026-04-28

---

## 一、项目背景

本项目是一个面向个人投资者和量化研究者的**股票数据分析平台**，通过 AI 对话驱动数据查询与可视化，支持将分析结果持久化到个人大屏 Dashboard。

项目同时作为简历作品，重点体现以下能力：
- LLM 接入与 Prompt 工程设计
- AI 驱动的数据可视化
- 前后端全栈开发能力

---

## 二、用户角色

| 角色 | 描述 |
|------|------|
| 普通用户 | 注册登录后使用系统，拥有独立的大屏和会话数据 |

---

## 三、功能需求

### 3.1 用户认证

- **注册**：输入用户名 + 密码完成注册，密码加密存储
- **登录**：用户名 + 密码登录，服务端签发 JWT Token
- **持久化登录**：Token 存储在本地，有效期内免登录
- **退出登录**：清除本地 Token

---

### 3.2 AI 聊天组件

AI 聊天是整个系统的核心交互入口，支持两种展示模式，用户可随时切换：

#### 展示模式
| 模式 | 描述 |
|------|------|
| 右侧固定侧边栏 | 聊天面板固定在页面右侧，与大屏并排显示 |
| 浮动气泡模式 | 大屏全屏展示，右下角浮动气泡按需展开聊天 |

#### 功能列表
- **流式输出**：基于 SSE 协议，AI 回复实时打字机效果展示
- **Markdown 渲染**：支持标题、列表、代码块、粗体等格式
- **多模态输入**：支持上传图片随消息一起发送
- **会话管理**：
  - 新建会话
  - 历史会话列表（持久化到数据库，按用户隔离）
  - 切换历史会话，恢复完整上下文
  - 删除会话
- **消息操作**：复制消息内容

#### AI 返回内容的特殊渲染
| 返回类型 | 渲染方式 | 操作 |
|------|------|------|
| ECharts 图表配置（JSON） | 内联渲染图表 | 显示"添加到大屏"按钮 |
| 表格数据（JSON） | 渲染高级 Table | 显示"添加到大屏"按钮 |
| 普通文本 | Markdown 渲染 | 无 |

---

### 3.3 大屏 Dashboard

#### 布局
- 基于 `react-grid-layout` 实现自由拖拽布局
- 每个组件（Widget）支持：
  - 拖拽移动位置
  - 拖拽边角缩放大小
  - 删除
  - 标题编辑

#### 组件类型
| 类型 | 数据来源 |
|------|------|
| ECharts 图表 | 来自聊天中 AI 返回的 ECharts option JSON |
| 数据表格 | 来自聊天中 AI 返回的表格数据 JSON |

#### 持久化
- 大屏布局（位置、尺寸）持久化到数据库
- 组件数据（ECharts 配置、表格数据）持久化到数据库
- 每个用户独立，登录后自动恢复上次大屏状态
- 实时自动保存（布局变更后触发）

#### 一键插入大屏
- 聊天中 AI 返回图表或表格时，消息底部显示 **"添加到大屏"** 按钮
- 点击后自动在大屏新增对应 Widget，默认尺寸插入到布局末尾

---

### 3.4 数据查询能力（由 nanobot 提供）

- 用户用自然语言提问，如："帮我查一下茅台最近一个月的收盘价走势"
- nanobot 自动生成 SQL 查询 MySQL 数据库，一步返回：
  - 查询结果数据
  - ECharts 可视化配置 或 表格数据结构
- 前端直接渲染，无需二次处理

## 四、非功能需求

| 项目 | 要求 |
|------|------|
| 响应速度 | 流式输出首字符延迟 < 2s |
| 数据安全 | 用户数据严格隔离，JWT 鉴权 |
| 可维护性 | 前后端分离，接口文档完整 |
| 部署方式 | 支持本地运行，Docker Compose 一键部署 |

---

## 五、数据库表设计（概览）

| 表名 | 用途 |
|------|------|
| `users` | 用户账号（用户名、密码哈希） |
| `conversations` | 会话列表（标题、用户ID、创建时间） |
| `messages` | 消息记录（角色、内容、会话ID） |
| `dashboard_widgets` | 大屏组件（类型、数据、布局信息、用户ID） |

---

## 六、不在范围内

- 用户管理后台（增删改查用户）
- 实时行情推送
- 量化回测引擎（展示分析结果即可）
- 权限分级（所有用户权限相同）
